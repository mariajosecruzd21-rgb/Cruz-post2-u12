# ADR-003: Spring Events (Observer) para notificaciones

## Estado: Aceptado — 2026-03-23

## Contexto
El servicio de pedidos necesita notificar a multiples receptores
(email, log, futuro SMS) cuando un pedido es procesado. El
acoplamiento directo a cada canal hace fragil el servicio central.

## Decision
Se usa Spring ApplicationEventPublisher (implementacion de Observer)
para publicar PedidoProcesadoEvent. Cada canal implementa un
@EventListener independiente.

## Consecuencias
Positivas:
- Agregar un nuevo canal no requiere modificar FachadaPedidos.
- Los listeners son opcionales: si no hay suscriptores, el evento
  se ignora sin error.
Negativas:
- El orden de ejecucion de listeners no esta garantizado.
- El debugging es mas complejo: hay que rastrear listeners para
  entender que ocurre tras publicar el evento.
